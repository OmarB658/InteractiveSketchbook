Current Build:

At the moment I have the basic structure of the project built. I have two circles that represent the lucky and unlucky player. They both start at the same size and position on opposite ends of the screen. When the user clicks their mouse, a random() function is triggered that determines which circle grows or shrinks. The lucky player has a 50% chance to grow, while the unlucky player has a 10% chance to grow but also a 5% chance to shrink completely. I think the gambling aspect works really well as you can see the visual gap between the circles grow over time, which creates a sense of frustration for the user as they see the unlucky player struggle to keep up with the lucky player.

What is the pressure:

What creates tension in my system is mostly accumulative with the way the circles grow and shrink over time. The user is constantly trying to get the lucky player to grow while also trying to prevent the unlucky player from shrinking.

Next week I plan to implement a decay feature that will make the circles shrink over time, which will add more pressure on the user to keep the lucky player growing and prevent the unlucky player from shrinking.
